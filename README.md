*Идея: Ефимов Кирилл*

## Концепция

1. Отсутствие ключевых слов. Все блоки и переходы обозначаются простыми символами.
2. Однозначная трансляция в графический вид.
3. Содержимое блоков выделяется отступами.

## Синтаксис

### Блоки и переходы

1. Именованный блок
```
[block]
  logic
```

2. Переход к именованному блоку
```
-> block
```

### Условия

1. Простейшее условие
```
<condition>
  logic if
```

2. Добавление "иначе"
```
<condition>
  logic if
-
  logic else
```

3. Знак + для обозначения блока условия
```
<condition>
+
  logic if
-
  logic else
```

### Циклы
Основаны на блоках, условиях и переходах.

1. do-while
```
[block]
  logic

<condition>
  -> block
```

2. Упрощённая форма do-while
```
[do]
  logic
<- <while>
```

3. while
```
[block_start]
<condition>
- -> block_end

logic
-> block_start

[block_end]
  logic
```

4. Упрощённая форма while
```
<while>
  logic
<-
```

5. for-each (TODO)
```
<item : list>
  logic
```

### Функции
Представляет собой отдельную блок-схему с началом и концом.

1. Объявление
```
(function)
  logic
```

2. Объявление с параметрами
```
(function : a, b, c)
  logic
```

3. Вызов
```
-> (function)
```

4. Вызов с параметрами (TODO)
```
-> (function : a, b, c)
```

### Комментарии

1. Однострочный комментарий
```
// comment
```

2. Многострочный комментарий
```
/* comment line 1
   comment line 2
   ...
   comment line n */
```

### Примеры

1. Сортировка пузырьком:

```
(sort : arr, n)
    //n = 10
    //arr = [4, 5, 8, 1, 5, 4, 3, 3, 2, 9]

    i = 0
    <i < n>
        j = 0
        <j < n - 1>
            <a[j] >= a[j+1]>
                tmp = arr[j]
                arr[j] = arr[j+1]
                arr[j+1] = tmp

            j = j + 1
        <-

        i = i + 1
    <-
```
